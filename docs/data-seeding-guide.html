Already loaded sample data The system comes pre-loaded with 10 sample products (Hawaii Classic, PU Sandal, EVA Slipper etc.) across 3 sections. You can view these in the Products page. Feel free to edit or deactivate them and add your real products.

Recommended data entry order

  1. List all your articles (e.g., Hawaii Classic, Hawaii Premium, PU Formal, etc.)
  2. For each article, list all available colours
  3. For each article + colour combination, list all available sizes
  4. Create one product entry for every Article + Colour + Size combination

3 Generate Child Box QR Labels

A Child Box represents one box of footwear (1 pair). Each child box gets a unique QR label that is printed and stuck on the box. This is the core scanning unit.

Where to go

Sidebar → Child Boxes → Click "Generate QR Labels"

Step-by-step

  1. Select Article — Pick the article from the searchable dropdown (type to search)
  2. Select Colour — Colour pills appear; click the one you want
  3. Enter Quantities — A size table appears. Enter the number of labels needed per size
    Example: If you have 50 boxes of Size 7, 80 boxes of Size 8, and 60 boxes of Size 9, enter those numbers in the respective rows.
  4. Review Summary — The live summary panel shows total labels and breakdown
  5. Click "Confirm & Generate" — Labels are created instantly
  6. Print Labels — Click "Print Labels" to open the print dialog. Use your TSC thermal printer (40×60mm label size)

4 Create & Pack Master Cartons

A Master Carton is a large shipping carton that holds multiple child boxes. You scan child box QR codes to "pack" them into a master carton.

Where to go

Sidebar → Master Cartons → Click "Create Carton"

Step-by-step

  1. Set Capacity — Default is 50 boxes per carton (adjust if needed, max 100)
  2. Scan Child Boxes — Use your phone camera or QR scanner to scan child box labels
    • Alternatively, type the barcode manually (e.g., BINNY-CB-xxxx) and press Enter
    • Each scan adds the box to the carton. You'll see product details (Article, Colour, Size, MRP)
  3. Review Scanned Items — The scanned items list shows all packed boxes with details
  4. Click "Create Master Carton" — The carton is created with status ACTIVE

Master Carton label

After creation, go to the carton's detail page and click "Print Label". The label includes:

  • Binny logo
  • Article, Colour, MRP
  • Size assortment grid (per-size quantities)
  • Total pairs
  • Pack date
  • QR code (BINNY-MC-{uuid})
Full-screen scan mode On mobile, tap the full-screen icon for a larger scanning area. The screen will stay awake during scanning.

Carton statuses

5 Close & Store Cartons

Once a carton is fully packed, close it for storage. This marks it as sealed and ready for dispatch.

Where to go

Sidebar → Storage

Step-by-step

  1. Scan the master carton QR code (or enter barcode manually)
  2. The carton details appear — review child boxes inside
  3. Click "Close & Store"
  4. Status changes from ACTIVE → CLOSED
Optional step You can dispatch ACTIVE cartons directly without closing them first. Closing is useful when cartons are prepared in advance and stored before dispatch.

6 Dispatch Cartons

Dispatch one or more master cartons to a customer. This is the final operational step.

Where to go

Sidebar → Dispatch

Step-by-step

  1. Select Customer — Pick from the dropdown (delivery location auto-fills)
  2. Scan Cartons — Scan master carton QR codes to add them to the dispatch
    • Only ACTIVE or CLOSED cartons can be dispatched
    • Product summary appears for each scanned carton
  3. Fill dispatch details (all optional):
    FieldExampleNotes
    DestinationJaipur WarehouseAuto-filled from customer
    Transport DetailsABC TransportTransport company name
    LR NumberLR-2026-0042Lorry Receipt number
    Vehicle NumberRJ-14-AB-1234Truck/vehicle registration
    NotesHandle with careAny additional instructions
  4. Click "Dispatch"

What happens on dispatch

  • Master carton status → DISPATCHED
  • All child boxes inside → DISPATCHED
  • Dispatch record is created with full audit trail
  • Stock levels update automatically in the Inventory dashboard

View dispatch history

Sidebar → Dispatches — shows all dispatches grouped by customer, with expand-to-see-details.

8 Check Inventory Dashboard

The Inventory page gives a real-time, drill-down view of your stock levels.

Where to go

Sidebar → Inventory

Features

  • Summary KPI cards — Pairs in Stock, Pairs Dispatched, Child Boxes, Active Cartons
  • Hierarchical drill-down — Click through: Section → Article → Colour → Size
  • Visual stock bars — Green (free/in stock), Blue (packed in carton), Gray (dispatched)
  • Breadcrumb navigation — Click any level to jump back
  • Refresh button — Reload latest data

Other Operations

Scan (Sidebar → Scan)

Quick barcode lookup page. Scan or type any barcode to see its details and current status. Useful for warehouse staff to check individual boxes.

Unpack (Sidebar → Unpack)

Remove ALL child boxes from a master carton. Scan the carton → Confirm → All boxes return to FREE status. The carton goes back to CREATED.

Repack (Sidebar → Repack)

Move child boxes from one carton to another. Scan source carton → Select boxes → Scan destination carton → Confirm. Useful when reorganizing shipments.

  UAT Testing Checklist

Walk through these items to validate all features during testing:

Setup & Master Data

  • Login with admin credentials
  • Add 3-5 real products (different articles, colours, sizes)
  • Add 2-3 real customers with GSTIN and delivery locations
  • Create additional users with different roles (Supervisor, Warehouse Operator, Dispatch Operator)

Core Workflow

  • Generate child box QR labels (try 5-10 per size)
  • Print labels and verify all information is correct on the printout
  • Create a master carton and scan/enter child box barcodes to pack
  • Print master carton label and verify size assortment grid
  • Close the carton via Storage page
  • Dispatch the carton to a customer
  • Verify dispatch appears in Dispatches list

Verification

  • Check Stock Report — numbers match what you created/dispatched
  • Check Carton Inventory — all cartons show correct status
  • Check Dispatch Report — dispatches show under correct customer
  • Export a CSV report and open in Excel
  • Trace a child box barcode — see full lifecycle timeline
  • Trace a master carton barcode — see all boxes inside
  • Check Inventory page — drill down Section → Article → Colour → Size

Role-Based Testing

  • Login as Warehouse Operator — verify can generate boxes & pack cartons but cannot see Products/Customers pages
  • Login as Dispatch Operator — verify can dispatch but cannot create boxes/cartons
  • Login as Supervisor — verify can manage products/customers but cannot delete users

Mobile / PWA

  • Open the app on a mobile phone
  • Install as PWA (tap "Install Binny Inventory" banner or browser's Add to Home Screen)
  • Test QR scanning using phone camera
  • Test full-screen scan mode
  • Turn off WiFi — verify offline indicator appears
  • Turn WiFi back on — verify "Back online" message
