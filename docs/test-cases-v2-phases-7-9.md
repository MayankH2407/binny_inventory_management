## Notes

- All API test cases require a valid JWT `Authorization: Bearer {token}` header. Tokens are obtained via `POST /api/v1/auth/login`.
- RBAC for reports is enforced at the router level via `router.use(authorize(USER_ROLES.ADMIN, USER_ROLES.SUPERVISOR))` — all report routes require Admin or Supervisor role.
- Inventory endpoints (`/api/v1/inventory/*`) have no RBAC role restriction beyond authentication — all authenticated users can access them.
- The trace endpoint is `GET /api/v1/inventory/trace/:barcode` (not a standalone `/trace` route).
- Daily Activity report (`/reports/daily-activity`) requires both `from_date` and `to_date` query parameters; omitting either returns HTTP 400.
- Barcode formats: Child boxes use `BINNY-CB-{uuid}`, master cartons use `BINNY-MC-{uuid}`.
- Dashboard `recentTransactions` returns the last 20 transactions ordered by `created_at DESC`.
- All response bodies follow the envelope format: `{ success: true, data: {...}, message: "..." }`.
